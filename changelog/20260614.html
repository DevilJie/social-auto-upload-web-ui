01 / CORE FEATURE

草稿批量发布

最重磅的新功能 — 选中多个草稿,一键入队,自动按平台/账号展开为多个发布任务。

  • 批量入队 + 任务溯源 每个 (草稿 × 账号) 自动展开为独立任务,记录 source/draft_id/account_id,方便追溯失败来源。
  • 预览弹窗 + 默认全选 弹窗显示标题、目标平台、状态,默认勾选所有"通过"草稿;支持表头一键全选/取消,失败行自动禁用。
  • 失败立即停止,不再重开浏览器 草稿批量发布任务 max_retries=0,失败直接标记 FAILED,不再因为用户关闭浏览器而反复重试。
  • 图文草稿也支持批量 独立的图文批量发布端点,与视频草稿按 tab 分流,共用弹窗组件体验一致。

支持的操作

  • 视频草稿批量发布
  • 图文草稿批量发布
  • 批量删除草稿
  • 校验失败的草稿不入队

02 / PLATFORM REWRITE

TikTok 全面重写

发布与登录流程整体重构,主题色回归品牌粉 #FE2C55,稳定性大幅提升。

  • 发布流程重写 重写整个 TikTok 发布流程,完善参数解析、文件设置、确认弹窗处理等关键环节。
  • 跨 frame 查找确认按钮 发布确认弹窗可能出现在 main 或 iframe 中,新逻辑遍历所有 frame 精确定位按钮。
  • 品牌主题色统一 TikTok 主题色从纯黑改为品牌粉 #FE2C55,与官方视觉保持一致。
  • 登录流程重写 同步重写登录流程,优化 Cookie 检测和会话恢复机制。

关键改进

  • set_input_files 超时 60s
  • page.is_closed() 安全退出
  • attached state 等待渲染
  • 品牌粉 #FE2C55
03 / HISTORY UPGRADE

发布历史详情页

从扁平列表升级为批次详情页 + 卡片网格,失败账号也保留完整内容快照。

  • 批次详情页 新增独立路由 /publish-history/:batchId,完整呈现一次"发布"的所有账号明细。
  • 4 列卡片网格 列表页从展开视图改为自适应卡片网格,封面 / 标题 / 状态 / 平台徽章一目了然。
  • 失败账号完整快照 失败账号也展示当时填写的内容(标题/描述/标签/封面),错误以红条叠加顶部,便于复盘。
  • 4 指标统计卡 + 渠道徽章 PublishStats 占位卡支持空数据,ChannelSummary 渠道徽章溢出时滚动展示。

04 / PERSONALIZATION

4 级合并发布配置

公共 → 平台默认 → 平台覆写 → 账号覆写,逐层精细控制,每个账号都能独立微调。

  • 4 级优先级合并 merge_config 实现 4 层合并,字段从公共配置逐层覆盖,逻辑清晰可预测。
  • 平台 / 账号级覆写区 PublishCenter 新增可视化覆写区,支持为每个平台或每个账号单独设置标题、描述、标签、声明等。
  • 草稿持久化覆写 platformOverrides / accountOverrides 一并保存到草稿,下次加载完整还原。
  • 校验 collect-all 校验错误一次性收集,单条通知显示所有问题,避免逐个修复反复弹窗。

合并优先级

  • 1. 公共配置(commonConfig)
  • 2. 平台默认(platformConfigs)
  • 3. 平台覆写(platformOverrides)
  • 4. 账号覆写(accountOverrides)
05 / UPLOAD & COMPONENTS

流式上传 + 统一组件

后端流式接收大文件,移除 1GB 限制;前端抽出统一 MaterialUploader 组件复用。

  • 后端流式上传 大文件上传改为流式接收,显著降低内存占用,提升上传稳定性。
  • 移除 1GB 大小限制 不再人为限制单文件 1GB,支持任意大小视频/图片上传。
  • 统一 MaterialUploader 组件 抽出通用上传组件,封面、视频、图文素材都走同一套交互。

带来的好处

  • 内存占用降低
  • 无大小限制
  • 上传交互统一
06 / PLATFORM STABILITY

多平台稳定性提升

B站、百家号、抖音、腾讯视频等多个平台的关键修复,让发布更可靠。

  • B站分区选择 分区下拉先按 section 定位再找 select-controller,加 has-text 模糊匹配 + force click 兜底,zone 字段支持中文名输入。
  • 百家号登录 修复登录超时自动关闭浏览器的问题,避免登录中断。
  • 抖音描述 Tag 注入 描述后按 Space 确保后续 tag 被识别为 hashtag;strip 描述尾随空格避免 tag 注入失败。
  • 腾讯视频表单等待 等 publish form 渲染完(attached 状态)再找 input,避免页面未就绪导致的元素丢失。
  • 单封面兜底 只上传一个封面时,另一封面自动用同图兜底,不再因为缺图卡住发布。

覆盖平台

  • B站 · 分区选择
  • 百家号 · 登录修复
  • 抖音 · Tag 注入
  • 腾讯视频 · 表单等待

07 / BACKEND ARCHITECTURE

双表 + asyncio 任务队列

引入 publish_batches / publish_details 双表模型,asyncio Worker 队列带完整溯源字段。

  • 双表数据模型 publish_batches(发布批次)+ publish_details(账号明细),一对多清晰建模,统计聚合更准确。
  • asyncio 任务队列 + Worker TaskQueue 基于内存 asyncio.Queue + 后台线程,支持并发控制、状态聚合、SSE 实时推送。
  • 任务溯源字段 PublishTask 扩展 source / draft_id / account_id / detail_id / payload 五字段,失败可精准定位。
  • 状态聚合修正 batch 聚合考虑 in-flight detail,避免误标 success;worker 防止 task_done 多次调用导致死锁。

架构亮点

  • SSE 实时推送
  • 并发数可配置
  • 状态变更回调
  • payload in-memory 透传
