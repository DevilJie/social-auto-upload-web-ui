01 / CORE · 支付宝内容创作中心

支付宝完整对接(视频 + 图集)

对接支付宝内容创作中心 c.alipay.com,支持视频发布、图集发布、合集搜索选择、音乐抽屉、作者声明、资料抓取同步等完整链路。24+ 个提交,历经 2 周迭代稳定。

  • 视频发布全流程 视频文件上传 → 表单填写(标题/描述/话题/封面/合集/作者声明) → 定时发布 → 确认发布。所有 antd5 + CSS Modules hash 类名都用 placeholder/role/text/aria/lable[title] 定位,稳定抗页面漂移。
  • 图集发布(图片/标题/描述/作者声明/音乐) 独立 short-content 发布页,多图上传、4 级合并配置、面板级 validate/getConfigs/publish 委托,各平台 panel 自定义校验逻辑互不干扰。
  • 合集功能(参考抖音音乐同款) 监听 queryCompilationsByPublicId.json,前端下拉搜索合集,按合集 title 选中(而非 ID,避免账号差异)。账号级设置也能正确生效(修复了 mergeConfig 字段遗漏)。
  • 音乐抽屉(精确网络拦截) 监听 queryAllMaterial.json 拿到音乐列表,客户端分页 + 任意 cookie 即可使用;支持翻页查找目标音乐,过期/失败都有兜底。
  • 作者声明(去掉 class 定位) 用 input[id$='_tagList'][role='combobox'] + [title='...'] 定位选项,失效自动跳过;新增「内容无需添加声明」选项,声明可选。
  • 资料同步与日志 scrape_alipay_profile 抓昵称 + 头像;独立的 alipay.log 日志文件 + 完整上送参数打印;「发布请注意」优化提示弹窗点继续发布即可跳过。

02 / SAFETY · Cookie 预检

发布前 Cookie 预检弹窗

点发布后立刻弹出,先并发检查所有选中账号的 cookie 状态;有失效账号则以扁平小卡片列出,用户逐个点击「重新登录」修复后才继续发布,杜绝「发到一半才提示失败」。

  • 触发时机 表单校验(标题/封面/视频格式等)全部通过后、正式调用渠道发布前触发,避免表单不合规时白跑检查浪费时间。
  • 并发检查 + 实时进度 前端限流 4 路并发调 /checkAccount,每个请求返回后立即更新对应账号卡片的状态(检查中 → 正常/失效),进度条 + 卡片状态实时刷新。
  • 扁平小卡片 失效账号以扁平卡片形式列出,红色左边框 + 平台 logo + 账号名 + 「重新登录」按钮。账号过多时自动出现垂直滚动条,弹窗不会撑爆。
  • 复用 SSE 登录流程 每张失效卡片点击「重新登录」后,通过 EventSource('/login?type=X&id=temp&account_id=N') 与现有后端 SSE 登录流程对接,登录成功自动从失效列表移除。
  • 同步 store 状态 检查/修复过程中同步更新 Pinia accountStore,让账号管理页也能即时反映账号状态变化。
  • 三阶段状态机 checking(进度条+实时卡片) → fixing(失效卡片+逐个修复) → done(绿色提示自动关闭);全部有效则跳过 fixing 直接进入 done。

ROOT CAUSE · 为什么需要 v1.0.0

从「能用」到「好用」的跨越

v0.9.x 让所有现有渠道稳定运行;v1.0.0 则是在此基础上接入完整的支付宝生态,并把整个发布体验打磨到「贴心」。

  • ① 渠道矩阵的最后一公里 v0.9.x 已支持抖音/快手/小红书/B站/百家号等主流平台,但支付宝作为重要渠道一直缺失。v1.0.0 完成支付宝内容创作中心的全流程对接,从平台注册到合集选择全部跑通。
  • ② 「发到一半才失败」的痛点 之前发布前不检查 cookie,经常跑到某个账号才弹「cookie 失效,请重新登录」,用户体验割裂。v1.0.0 新增预检弹窗,发布前一次性把失效账号摆出来逐个修复,通过后才继续发布。
  • ③ 个性化模式长期误判 草稿支持每个账号上传不同视频(个性化覆写),但发布校验只看 commonConfig,导致非通用模式一律被误报「请上传视频」。v1.0.0 修复校验逻辑扫三源。
  • ④ 新平台映射表经常漏改 之前新增平台(头条)时,后端 3 处平台 id→key 映射表全部漏改,导致头条账号在草稿箱不显示。v1.0.0 改为列表查询时实时重算,后续新增平台也不会复发。
  • ✓ 后续规划 v1.0.x 系列会持续打磨支付宝和头条的边角案例,同时预检机制会扩展到「素材有效性」「定时发布时间冲突」等更多发布前检查项。
