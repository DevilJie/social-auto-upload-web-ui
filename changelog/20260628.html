01 / CORE · 小红书

小红书全能力打通

内容来源声明、合集、话题校验、描述预处理、封面方向、标题校验 — 一次性补齐小红书所有缺失能力。

  • 内容来源声明(自主拍摄 + 来源转载) 一级选「内容来源声明」→ 二级选「自主拍摄」(弹窗填拍摄地点 POI 搜索+拍摄日期) 或「来源转载」(填转载来源)。选来源转载时原创声明自动禁用并还原为非原创。
  • 合集功能(账号级配置) 新建 xiaohongshu_bp.py,打开发布页 → 点「加入合集」→ 解析 item-label DOM 拿合集列表。前端账号级合集下拉,发布时按合集名称自动选择。
  • 话题 #xxx 总数 ≤ 10 前置校验 描述里独立 #xxx + 标签数 ≤ 10。仿抖音实现,_HASHTAG_PATTERN 正则同语义。前后端双重校验,含单元测试(19 个用例)。
  • 描述预处理 + 标签分隔 描述自带 #xxx 自动补空格(确保小红书识别成话题);描述末尾加空格分隔后续标签;输入加 delay 30ms 模拟真人。
  • 封面按视频方向自动选择 读取素材 orientation 字段:横版视频→横版封面,竖版/正方→竖版封面。取代原来硬编码「竖版优先」。
  • 标题 ≤ 20 字校验 复用已有 validate_title_for_platform(emoji 按 3 算),前后端双重校验。

小红书能力清单

  • 话题 ≤10 校验
  • 描述 #xxx 补空格
  • 封面横竖版匹配

02 / CORE · B 站

B 站上传等待重构 + 合集 + 校验

重写上传完成判断逻辑,去除所有 class 定位;新增合集功能;标题简介前后置校验。

  • 上传完成判断重写 唯一标志:「上传完成」文案(get_by_text)。去除 div.cover-main/.bp-upload-progress 等 class 定位。超时 4 小时,0.5s 轮询。表单就绪改用 input[placeholder*="标题"]。
  • 标题 ≤ 80 字 + 禁止 emoji 新增 _sanitize_title() 黑名单过滤:去掉 emoji(非 BMP 字符)+HTML 危险字符(<>"'&),保留中文/英文/数字/常见标点。截断 80 字。
  • 简介 ≤ 2000 字(emoji=3) 新增 _truncate_desc_by_length() 按 emoji=3 规则截断。新增 validate_desc_for_platform() 后端校验。
  • 合集功能(有头模式→无头) 新建 bilibili_bp.py,打开上传页 → 上传视频触发页面跳转 → 等标题输入框 → 点「请选择合集」→ 解析 season-item-title DOM。开发阶段有头模式验证通过后切无头。

B 站能力清单

  • 上传完成判断(4h 超时)
  • 标题 ≤80 + 禁 emoji
  • 简介 ≤2000(emoji=3)
  • 合集选择

05 / UNIFY · 跨平台 + 设置

跨平台输入统一 + 系统设置增强

11 个渠道输入清空逻辑统一;系统设置新增账号检查机制;修复自动填充标题开关。

  • 跨平台输入清空统一 新增 _utils.clear_input() / clear_and_type() helper。Mac 用 Meta+A(Cmd+A),其他用 Control+A。input 用 fill("") 清空,contenteditable 用全选+Delete。11 个平台全部改用。
  • 百家号描述清空修复 描述输入前先清空编辑器(Ctrl+A+Delete),避免残留视频素材 ID 等随机字符串。
  • 自动填充标题开关修复 Settings.vue handleSave 只同步了封面比例到 store,漏掉了 autoFillTitle/autoSaveDraft/autoSaveInterval。补上三项同步。
  • 账号登录状态检查机制 新增设置项,下拉选择:「项目启动时后台检测」/「发布前检测」。两个机制互斥只能生效一个。启动时检测:后端异步遍历所有账号调用 check_cookie,更新 status(1=有效/0=无效)。

影响的平台

已统一 clear_and_type 的平台: 小红书 · 抖音 · B 站 · 视频号 快手 · 百家号 · 爱奇艺 · 腾讯视频 TikTok · 支付宝 · 今日头条 · 微博 Mac 兼容: Meta+A → Control+A 自动切换
ROOT CAUSE · 为什么需要 v1.1.0

从「能发布」到「发得好」的跨越

v1.0.0 让支付宝和头条跑通;v1.1.0 则是在此基础上把每个渠道的发布体验打磨到「靠谱」。

  • ① 小红书能力长期缺失 之前小红书没有合集、内容来源声明、话题校验等能力,用户需要手动在小红书 APP 操作。v1.1.0 一次性补齐,自动化覆盖率从 60% 提升到 95%。
  • ② B 站上传等待逻辑脆弱 之前用 class 定位(div.cover-main 等)判断上传完成,B 站前端一更新就失效。v1.1.0 改用「上传完成」文案定位+4 小时超时,稳定抗页面漂移。
  • ③ MacOS 输入清空失效 所有平台硬编码 Control+KeyA,MacOS 上应该是 Meta+KeyA(Cmd+A)。v1.1.0 统一用跨平台 helper,自动检测系统。
  • ④ 设置开关不生效 「自动填充标题」开关关闭后仍生效——Settings.vue 保存时漏同步到 Pinia store/localStorage。v1.1.0 修复。
  • ✓ 后续规划 v1.1.x 系列会持续打磨知乎和视频号的边角案例,同时扩展「启动时检测」到更多检查项(素材有效性、定时发布时间冲突等)。
