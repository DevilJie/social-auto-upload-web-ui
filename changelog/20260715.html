v1.2.1

亮色主题来了 · 所见即所得

v1.2.1 是一次主题焕新版本 —— 新增完整亮色主题,右上角一键切换、本地持久化;51 个组件、420+ 处硬编码颜色统一替换为主题感知变量。本次连这条更新日志本身都以亮色主题渲染,就是这次的最好证明。同时修复发布成功误判失败、视频号/抖音登录稳定性与图集发布预检误弹等问题。

🌙 深色 ☀️ 亮色
2 套主题 51 组件适配 5 个提交 420+ 颜色替换

核心亮点

四大方向,亮色主题落地 + 登录/发布稳定性提升

完整亮色主题

styles/themes.scss 新增 dark/light 两套 CSS 变量(背景/文字/边框/叠加层/阴影),variables.scss 把 SCSS 变量映射到 CSS 变量,运行时实时切换;design-system.scss 的 Element Plus 变量也分两套。右上角图标按钮一键切换,localStorage 持久化记忆。

核心特性

51 组件主题感知改造

对 51 个 .vue 组件机械整改 420+ 处硬编码颜色(写死的 #fff / rgba(0,0,0,0.3) 等)统一替换为 $text-primary / $bg-surface 等主题感知变量。亮色下不再出现文字发白看不清、卡片底色发灰等残留问题。

体验

登录稳定性优化

视频号登录去掉 30s 超时的二维码提取逻辑(它是登录页被自动关闭的元凶),改为打开 /login.html + 轮询 URL 判断成功;抖音登录改为纯 URL 变化监听。彻底消除"扫了码却被弹回登录"的体感问题。

稳定性

发布成功误判修复

_browser.py 包装 browser.close 并新增 disarm 标志:发布成功后收尾自动关浏览器不再被误判为失败,而用户手动关浏览器仍按原逻辑正常判失败。一改之前"明明成功了却报红"的尴尬。

Bug 修复

详细功能

主题改造的具体落地与配套修复

✓

主题系统三层架构

themes.scss 定义 html[data-theme="dark"] / [data-theme="light"] 两套 CSS 变量; variables.scss 把 $bg-base / $text-primary 等映射到 CSS 变量,实现运行时切换; design-system.scss 覆盖 Element Plus 的 --el-bg-color / --el-text-color-primary 等,保证弹窗、表格、按钮全主题一致。

✓

主题切换 + 持久化

stores/app.js 新增 theme 状态与 toggleTheme() action,写入 localStorage('theme') 并同步设置 document.documentElement.dataset.theme。 App.vue 右上角渲染图标按钮(☀️/🌙), main.js 启动时从 localStorage 读回主题,避免刷新闪一下默认深色。

✓

硬编码颜色全面主题感知

典型场景: BatchTagDialog 账号选中态文字由写死 #fff 改为 $text-primary; CoverCard 封面比例标签文字固定 #fff、预览区背景由写死 rgba(0,0,0,0.3) 改为 $bg-surface; AccountManagement 标题渐变改用 $gradient-brand 全站统一、筛选按钮选中态文字由 #fff 改为 $brand-start。亮色下文字清晰可辨。

✓

标签弹窗中文输入修复

TagPopover 在 IME 组合输入(compositionstart → compositionend)期间屏蔽 popover 的关闭逻辑,修复中文选词时弹层意外消失、输入中断的老问题。拼音/双拼用户终于能顺畅打标签。

✓

视频号 / 抖音登录重写

视频号 login 去掉 _extract_qrcode_src 等 4 个二维码提取函数(其 30s 超时是登录页被自动关的元凶), 改为打开 /login.html + 轮询 URL(跳到 /platform/* 即成功); _is_login_completed 改为纯 URL 判断,去掉 has-text 误判; scrape_tencent_profile 用 wait_for(finder-card) 取代固定 sleep。 抖音 login 去掉多余 get_by_role(二维码).get_attribute, 改为 url_changed_event 无超时监听。
