详细功能

VIVO 平台实现细节 + 封面系统 + 账号运营数据

✓

VIVO 平台完整对接

创作者中心 https://www.kaixinkan.com.cn/#/home,视频发布 /#/content/uploads。规范:视频大小≤2G、时长≤90min、描述≤500字。完整流程:扫码登录 → 资料同步(粉丝/获赞) → 上传视频(4h 超时) → 描述+标签(#xxx 末尾空格激活话题) → 3:4 竖版封面 → 位置(浏览器自动化搜索) → 作品同步 → 自主声明 → 谁可以看/下载权限 → 定时发布 → 提交 → URL 跳转判定成功。

✓

VIVO 位置搜索自动化

/api/vivo/search-position:浏览器自动化打开 VIVO 发布页 → 上传测试视频触发表单 → 在 .sel-position-module 输入关键词 → 解析 .position-list li 的 .position-name + .position-info 返回,前端 VivoPositionSelect.vue 复用模式与小红书 POI 一致,空值即不显示位置。

✓

账号运营数据 5 元组兼容

user_info 迁移新增 fans/likes/follows 三列(默认 0)。BasePlatform.sync_profile 抽象方法约定:旧平台返回 2 元组 (name, avatar);VIVO 等新平台返回 5 元组 (name, avatar, fans, likes, follows)。save_login_result / syncProfile 路由 / setAccounts store 全部按元组长度兼容解包。前端账号卡片底部展示「粉丝 N · 获赞 N · 关注 N」,同步触发时立即更新。

✓

封面系统不再污染素材库

封面裁剪结果改为临时处理、不入素材库;新增素材库批量删除能力;缓存清理覆盖封面目录。封面弹窗尺寸 tab 改造 + 布局重设计 + 亮色样式修复。封面裁剪改为两个独立面板,确认时统一校验裁剪结果。

✓

浏览器关闭智能识别

_browser.py 包装 browser.close 并新增 disarm 标志:发布成功后的收尾关浏览器调用先 disarm,不再被监听器误判为「浏览器异常关闭 → 发布失败」。用户手动关浏览器时 disarm 未置位,仍按原逻辑正常判失败。app.py 发布时识别浏览器关闭异常,返回友好提示而非栈跟踪。

✓

视频号 / 抖音登录重写

视频号 login 改为打开 /login.html + 轮询 URL(跳到 /platform/* 即成功);_is_login_completed 改为纯 URL 判断;新增「视频标注」下拉(拍摄地点/转载来源联动);封面遍历所有入口设置 + 横版封面 popover 处理。抖音 login 改为 URL 变化监听;定时发布改用 Semi 时间滚轮选时/分,修复时间丢失 bug。
