# 封面编辑器重设计

> 借鉴主流视频平台（B站、抖音、YouTube）的封面设置交互，重新设计发布界面的封面编辑体验。

---

## 背景与问题

当前封面设置完全依赖手动上传图片，不支持从视频中截取帧作为封面，交互流程与主流平台差距较大。用户需要：

1. 从视频中自动/手动选取帧作为封面
2. 在弹窗内完成裁剪、上传、素材库选择等所有操作
3. 横版（16:9）和竖版（9:16）封面独立编辑

---

## 方案选型

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A: 后端 ffmpeg 抽帧 (选定)** | 视频上传后后端用 ffmpeg 预抽帧，前端请求帧图片 | 质量高、格式全兼容、前端零依赖 | 需内置 ffmpeg 二进制 |
| B: 前端 Canvas 抽帧 | 浏览器端用 video+canvas 抽帧 | 不依赖后端工具 | 格式兼容差、性能不可控 |

---

## 1. ffmpeg 内置与打包

### 二进制管理

- ffmpeg/ffprobe 静态编译二进制存放在 `backend/bin/` 目录
- 后端通过路径优先级定位 ffmpeg：
  1. PyInstaller 打包环境：`sys._MEIPASS/bin/ffmpeg`
  2. 开发环境：`backend/bin/ffmpeg`
  3. 兜底：系统 PATH

### exe 打包

- PyInstaller `.spec` 文件添加：`datas=[('bin/ffmpeg', 'bin'), ('bin/ffprobe', 'bin')]`
- ffmpeg 静态编译版约 70-100MB，需评估对打包体积的影响
- 调用方式使用 `subprocess` 而非 `ffmpeg-python`，减少依赖链

---

## 2. 后端抽帧 API

### 抽帧触发

视频上传完成后立即异步执行，不阻塞上传接口响应。

### API 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/extract-frames` | POST | 触发抽帧任务 |
| `/api/frames-status` | GET | 查询抽帧进度 |
| `/api/frames` | GET | 获取已抽帧列表 |
| `/api/frame-image` | GET | 获取指定帧高清图 |

### 详细定义

**POST /api/extract-frames**
- 参数：`{ video_path: str, video_type: 'landscape'|'portrait' }`
- ffmpeg 命令：`ffmpeg -i {video} -vf fps=1,scale=320:-1 -q:v 3 {output_dir}/frame_%d.jpg`
- 输出目录：`{BASE_DIR}/frames/{video_uuid}/`
- 每帧命名：`frame_{seconds}.jpg`（frame_0.jpg, frame_1.jpg...）
- 缩略图宽度固定 320px
- 返回：`{ task_id: str }`

**GET /api/frames-status?task_id=xxx**
- 返回：`{ status: 'processing'|'done', total_frames: int, video_duration: float }`

**GET /api/frames?video_path=xxx**
- 返回：`{ frames: [{ url: str, seconds: int }], duration: float }`
- 前端用于渲染时间轴和推荐帧

**GET /api/frame-image?video_path=xxx&seconds=5**
- 返回指定秒数的高清帧图片（原始分辨率）
- 有缓存直接返回，无缓存用 ffmpeg 实时截取

### 视频时长获取

`ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1 {video}`

## 3. 发布页封面卡片交互

### 空状态（视频已上传但未设封面）

1. 抽帧完成后，取均匀分布的 **6 帧**作为推荐帧（首帧 + 中间4帧 + 末帧）
2. 封面卡片显示 6 帧缩略图横排，点击任意帧直接设为封面
3. 右侧显示"编辑封面"按钮卡片，点击打开编辑弹窗
4. 底部保留"上传图片"和"从素材库选择"快捷入口

```
┌────────────────────────────────────────────────┐
│ 横版封面                                       │
│ ┌─────┐┌─────┐┌─────┐┌─────┐┌─────┐┌──────┐   │
│ │ ✓   ││     ││     ││     ││     ││ 编辑  │   │
│ │ 帧1 ││ 帧2 ││ 帧3 ││ 帧4 ││ 帧5 ││ 封面  │   │
│ └─────┘└─────┘└─────┘└─────┘└─────┘└──────┘   │
│ [上传图片]  [从素材库选择]                       │
└────────────────────────────────────────────────┘
```

### 已设封面状态

- 显示当前封面预览大图
- 覆盖层：编辑、替换、删除按钮
- 保留推荐帧横条，可快速切换

### 视频类型关联

- 只上传横版视频 → 横版封面从横版视频抽帧，竖版封面也从横版视频抽帧（无竖版源）
- 上传横版+竖版视频 → 各自使用对应视频抽帧
- 只上传竖版视频 → 同理

### 推荐帧选取

- 从已缓存帧中均匀选取6帧，不额外生成
- 前端加载封面区域时调用 `/api/frames` 获取

## 4. 封面编辑弹窗

### 整体布局

```
┌──────────────────────────────────────────────────┐
│  编辑封面                  [横版 16:9] [竖版 9:16] │
│──────────────────────────────────────────────────│
│  ┌──────────────────────────┐  ┌──────────────┐  │
│  │                          │  │  素材库       │  │
│  │     预览 / 裁剪区         │  │  ┌──┐┌──┐   │  │
│  │     (16:9 或 9:16)       │  │  └──┘└──┘   │  │
│  │                          │  │  ┌──┐┌──┐   │  │
│  │  ┌─────────────────┐     │  │  └──┘└──┘   │  │
│  │  │  裁剪选区        │     │  │  ...        │  │
│  │  └─────────────────┘     │  │              │  │
│  │──────────────────────────│  │              │  │
│  │  ◄ ████████●████████ ►   │  │              │  │
│  │  时间轴 (缩略图条)        │  │              │  │
│  │  [上传图片]              │  └──────────────┘  │
│  └──────────────────────────┘                    │
│                                  [取消] [确认]    │
└──────────────────────────────────────────────────┘
```

### 左侧主区域

**预览/裁剪区：**
- 显示当前选中帧或图片
- 叠加裁剪框，比例锁定（横版 16:9，竖版 9:16）
- 支持拖动移动、角点缩放
- 复用现有 canvas 裁剪逻辑，适配新比例

**时间轴：**
- 横向缩略图条，每秒1帧采样
- 拖动滑块选帧，吸附到整数秒
- 选中帧实时更新预览区
- 底部时间标记（00:00, 00:10, 00:20...）

**上传按钮：**
- 时间轴下方"上传图片"，选择本地文件加载到预览区

### 右侧素材库

- 内嵌网格缩略图，滚动浏览
- 点击素材直接加载到预览/裁剪区
- 只显示图片类型素材
- 分页加载

### Tab 切换横版/竖版

- 切换时：裁剪比例变更（16:9 ↔ 9:16）、裁剪框重置
- 时间轴和素材库保持不变
- 横版和竖版各自独立保存选中图片和裁剪状态

### 确认逻辑

- 点击确认：裁剪框内区域导出为封面，上传后端
- 导出分辨率：横版 1920x1080，竖版 1080x1920
- 裁剪后重新上传到 `/upload` 端点，走现有文件存储流程

---

## 5. 数据流

```
视频上传 → 后端异步抽帧(ffmpeg) → 缓存帧缩略图(320px)
                                        ↓
发布页加载 ← GET /api/frames ← 帧列表(6个推荐帧)
                                        ↓
用户点击推荐帧 → 直接设为封面
用户点击编辑封面 → 打开弹窗
    ↓
弹窗内:
  时间轴拖动 → GET /api/frame-image → 预览/裁剪
  上传图片 → 加载到预览/裁剪
  素材库点击 → 加载到预览/裁剪
    ↓
确认裁剪 → canvas 导出 → POST /upload → 更新封面数据
```

---

## 6. 裁剪比例变更

当前竖版封面比例是 3:4，改为 9:16 以匹配主流短视频平台标准。

| 类型 | 当前比例 | 新比例 |
|------|---------|--------|
| 横版 | 16:9 | 16:9（不变） |
| 竖版 | 3:4 | **9:16** |

需同步检查各平台后端上传逻辑是否受影响。

---

## 7. 存储与性能

- 抽帧缩略图（320px）每帧约 10-20KB，1分钟视频约 60帧 ≈ 0.6-1.2MB
- 高清帧按需生成，可缓存
- 帧缓存目录：`{BASE_DIR}/frames/{video_uuid}/`
- 可定期清理已发布任务的帧缓存
